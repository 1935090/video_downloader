Install dependencies

Go to the source code folder, install node modules by running

$ npm install

Link native dependencies

React Native modules that include native Objective-C, Swift, Java, or Kotlin code have to be "linked" so that the compiler knows to include them in the app.

$ react-native link

Environment Configuration

Make sure you have set up your development environtment as indicated in The Official Docs of React Native. We will review a few important points in that as follows

Android

1. Install Android Studio: Make sure you have all of the following checked when installing Android Studio:

Android SDK

Android SDK Platform

Performance (Intel ® HAXM) (See here for AMD)

Android Virtual Device

2. Install the Android SDK:

Building a React Native app with native code requires the Android 9 (Pie) SDK

3. Configure the ANDROID_HOME environment variable

Add the following lines to your $HOME/.bash_profile or $HOME/.bashrc config file:

export ANDROID_HOME=$HOME/Library/Android/sdk

export PATH=$PATH:$ANDROID_HOME/emulator

export PATH=$PATH:$ANDROID_HOME/tools

export PATH=$PATH:$ANDROID_HOME/tools/bin

export PATH=$PATH:$ANDROID_HOME/platform-tools

export JAVA_HOME=/Library/Java/JavaVirtualMachines/adoptopenjdk-8.jdk/Contents/Home

Make sure all the links to sdk and java home above are pointed to correct locations on your machine. Then type source $HOME/.bash_profile to load the config into your current shell. Verify that ANDROID_HOME has been added to your path by running echo $PATH.

IOS

For iOS, in the ios/ directory run the following command to install Cocoa libs

$ pod install

Firebase/Admob configuration

All common configuration related to Firebase Admob have been done for you as indicated in https://rnfirebase.io. For configuratons specific to your firebase/admob account, you have to do some editing to the source code as guided below: (instructions taken from the official doc)

Android

To allow the Android app to securely connect to your Firebase project, a configuration file must be downloaded and added to your project.

On the Firebase console, add a new Android application and enter your projects details. Download the google-services.json file and place it inside of your project at the following location: /android/app/google-services.json.

IOS

To allow the iOS app to securely connect to your Firebase project, a configuration file must be downloaded and added to your project.

On the Firebase console, add a new iOS application and enter your projects details. The "iOS bundle ID" must match your local project bundle ID. The bundle ID can be found within the "General" tab when opening the project with Xcode. Download the GoogleService-Info.plist file.

Using Xcode, open the projects /ios/{projectName}.xcodeproj file (or /ios/{projectName}.xcworkspace if using Pods). Right click on the project name and "Add files" to the project, as demonstrated below:

Select the downloaded GoogleService-Info.plist file from your computer, and ensure the "Copy items if needed" checkbox is enabled.

Setting up Google AdMob

As indicated in https://rnfirebase.io/admob/usage, before you are able to display ads to your users, you must have a Google AdMob account. Under the "Apps" menu item, create or choose an existing Android/iOS app. Each app platform exposes a unique account ID which needs to be added to the project.

Within the root of your React Native project, locate firebase.json file, and change admob_android_app_id and admob_ios_app_id to your unique account IDs

For each App you created on Google Admob Console, you will also have to create "Ad Unit ID" for each ads you want to display, in App/Container/Downloader/DownloaderScreen.js or App/Containers/SavedVideo/SavedVideoScreen.js or wherever you want show ads, change the "bannerAdUnitId" to your own "Ad Unit ID". That's all for setting up Google AdMob.

Usage

To run the app on android, run

$ react-native run-android

To run the app on iOS

$ react-native run-ios

or

open `ios/FacebookVideoDownloader.xcworkspace` (instead of `ios/FacebookVideoDownloader.xcodeproj` in Xcode as we use pods)

More references on how to run React Native app on different platforms at React Native Docs
